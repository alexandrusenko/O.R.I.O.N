# O.R.I.O.N.

O.R.I.O.N. (Omni-Resourceful Intelligent Operations Network) — локальный AI-ассистент с инструментами, памятью и агентной логикой на LangChain.

## Что реализовано

- Модульная архитектура (`core`, `tools`, `memory`, `safety`, `ui`, `config`).
- Агент на `langchain` через `create_react_agent` (LangGraph prebuilt)..
- Динамическая загрузка инструментов из `orion/tools` с перезагрузкой во время работы.
- Инструменты: `web_search`, `web_content_load`, `create_file`, `exec_command`, `get_system_info`.
- Принудительная политика веб-поиска для запросов с актуальными данными (погода/новости/курсы и т.п.).
- Встроенное поведение на уточнение запроса: если данных не хватает, агент задаёт пользователю уточняющий вопрос.
- Память:
  - STM в SQLite (лог диалога),
  - LTM в SQLite с retrieval по семантической близости.
- Интерактивный терминальный интерфейс на Rich + prompt_toolkit с трейсингом tool-вызовов.

## Логика ответа

1. Агент получает пользовательский запрос + контекст из STM/LTM.
2. Если нужен внешний факт, агент вызывает веб-инструменты и читает контент страниц.
3. После выполнения tools возвращает краткую и качественную сводку.
4. При неоднозначности или нехватке данных агент сначала запрашивает уточнение у пользователя.

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .
python -m orion.app
```

Команды выхода: `exit` / `quit`.
Перезагрузка инструментов на лету: `Обнови свои инструменты`.

## LM Studio

Базовый endpoint: `http://localhost:1234/v1`.
Модель и runtime-настройки задаются в `orion/config/settings.py`.
